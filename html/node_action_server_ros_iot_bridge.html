node_action_server_ros_iot_bridge module¶

ROS Node - Action Server - IoT ROS Bridge

class node_action_server_ros_iot_bridge.IotRosBridgeActionServer¶

Bases: object

This class is responsible for all action server related work It will handle the goals sent by the action client or MQTT and process it.

mqtt_sub_callback(client, userdata, message)¶

This is a callback function for MQTT Subscriptions

Parameters:
  • client – This is the client which has sent the goal
  • userdata – The information about the action client
  • message – The message which is being published on MQTT
on_cancel(goal_handle)¶

This function will be called when Goal Cancel request is send to the Action Server

on_goal(goal_handle)¶

This function will be called when Action Server receives a Goal

Parameters:goal_handle – The goal sent by the action client will reside in this
process_goal(goal_handle)¶

This function is called in a separate thread to process Goal. For publishing or getting messages from MQTT this function is used.

Parameters:goal_handle – The goal sent by the action client will reside in this
process_goal_1(goal_handle)¶

This function is called in a separate thread to process Goal. For sending data to spreadsheet this function is used

Parameters:goal_handle – The goal sent by the action client will reside in this
node_action_server_ros_iot_bridge.main()¶

The driver function that will start the action server to recieve goals
